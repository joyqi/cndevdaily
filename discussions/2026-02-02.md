### 文章摘要

**The Disconnected Git Workflow**
> 本文介绍了一种**离线、多邮箱账号的 Git 工作流**，通过 `git send-email` 配合 `msmtp` 取代 GitHub 繁琐的网页 PR 流程，实现补丁提交、评论回复全部在本地完成，并支持针对不同项目自动选择发件账号与别名。作者详细说明了 `.msmtprc` 和 `.gitconfig` 的配置方法，以及如何利用 `msmtp` 的队列脚本在无网络时暂存邮件，提升开源协作的效率与隐私保护。技术价值在于**跨项目多身份管理的自动化**与**离线补丁提交流程的优化**，兼顾安全性和反垃圾邮件能力。

**Compiling Rust to readable C with Eurydice**
> Eurydice 是 Aeneas 项目的一部分，旨在将 Rust 代码编译为结构清晰、可读性高的 C 代码，从而兼容现有的 C 语言验证与合规工具，特别适合高安全性软件和没有 Rust 编译器的环境。它在保留原有代码结构和求值顺序的同时，移除 C 不支持的 Rust 特性，并通过支持代码解决迭代器等差异，已成功用于将后量子密码算法的 Rust 实现转换为可验证的 C 版本。这种方法为 Rust 在形式验证和跨语言迁移方面提供了技术桥梁和创新路径。

**4x faster network file sync with rclone (vs rsync) (2025)**
> 文章通过对比 **rsync** 与 **rclone** 在局域网文件同步中的性能，发现 rsync 因单线程传输限制，速度仅约 350 MB/s，59 GB 数据需 8 分钟以上完成；而 rclone 的 **--multi-thread-streams** 并行传输选项能显著提升速度，在同样环境下实现约 **4 倍加速**。技术创新点在于将 rclone 应用于本地 LAN 文件同步，并利用多线程流打破单文件串行复制瓶颈，大幅提升高带宽网络下的大文件批量同步效率。
