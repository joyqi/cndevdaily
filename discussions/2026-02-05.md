## 第二轮讨论：内容评选

### 文章摘要

**Sukr: A minimal static site compiler in Rust with zero-JS output**
> **总结：**  
Sukr 是一个用 Rust 编写的极简静态网站编译器，生成零 JavaScript 的高性能 HTML，避免臃肿的前端运行时。它的技术亮点包括基于 Tree-sitter 的精准语法高亮（支持语言内嵌）、构建时数学公式渲染（KaTeX）、构建时图表编译（Mermaid → SVG）以及无需重新编译的 Tera 模板支持，并可通过配置文件轻松管理多站点。该工具特别适合追求加载速度快、可维护性高且不依赖客户端脚本的静态网站开发场景。

**Taming the Flat AST: Ergonomics in the Age of Zero Allocations**
> 文章介绍了 Go 语言 LL(1) 解析器生成器 **egg** 的核心创新——将整个 AST 压缩为单个 `[]int32` 的“平面”内存布局，实现零堆分配和高 CPU 缓存命中率。作者通过引入轻量级 **view 结构**与 Go 1.23 **迭代器**，在保持性能优势的同时，大幅提升 AST 遍历的可读性与开发体验。这种方法隐藏了底层指针运算，使复杂的语法解析代码能以接近传统树形 AST 的方式编写，兼顾性能与可维护性。

**Sqldef: Idempotent schema management tool for MySQL, PostgreSQL, SQLite**
> Sqldef 是一个支持多种主流数据库的命令行工具，可对比两个 SQL schema 并生成差异对应的 DDL，实现幂等化的数据库结构管理。它的创新点在于直接使用标准 SQL DDL 进行迁移，并提供基于 WebAssembly 的在线演示，方便快速验证和生成变更脚本。
