### 文章摘要

**GitHub Actions Is Slowly Killing Your Engineering Team**
> 作者基于丰富的 CI 系统使用经验，指出 GitHub Actions 虽然因与代码库集成方便而占据市场，但在核心体验上存在严重缺陷，尤其是日志查看与调试流程低效、耗时且易崩溃。文章强调优秀 CI 应该减少繁琐配置和无谓的操作开销，并举例 Buildkite 和 Nix 生态下的 Garnix，展现更高效、即用即走的 CI 设计理念。

**Owning a $5M data center**
> 文章介绍了公司在办公室自建约 500 万美元数据中心的经验与优势。作者强调，相比云服务，自建数据中心能降低长期成本、提升工程能力，并促使机器学习研发团队优化代码与算法而非单纯增加算力。技术创新点包括：利用外部空气和 PID 控制实现高效低功耗冷却、自主研发可维护的高密度 GPU 服务器，以及面向稳定算力需求的存储架构设计。

**Building a C compiler with a team of parallel Claudes**
> 作者提出了一种称为“agent teams”的新方法，让多个 Claude 实例在没有人工干预的情况下并行协作开发同一代码库，并成功用 16 个并行代理从零编写出可编译 Linux 内核的 Rust C 编译器。技术创新在于构建了长时间运行的自动化 harness，通过任务分解与文件锁机制实现多代理协作、并行调试和专业分工，同时探索了合并冲突处理与无人监督下保持进度的策略，为大规模自主 LLM 代码生成提供了可行的实验原型。
