## 第二轮讨论：内容评选

### 文章摘要

**An Open Letter to Google regarding Mandatory Developer Registration for Android**
> 文章反对谷歌即将实施的 **强制开发者注册政策**，该政策要求所有 Android 应用开发者在向 Google Play 之外的渠道分发应用前，必须向谷歌注册并提交身份信息。作者认为此举打破了 Android 的开放生态，扩大了谷歌在第三方渠道的控制权，引入额外的许可和成本，阻碍小型、开源、隐私导向及特殊领域开发者的创新，同时带来数据集中和隐私风险。文章呼吁谷歌与开源和安全社区合作，寻找更开放、更不具限制性的替代方案。

**Making WebAssembly a first-class language on the Web**
> 文章介绍了 WebAssembly 自 2017 年首次发布以来的功能演进，从仅适配 C/C++ 到支持更多语言，通过新增 SIMD、GC 支持、多返回值等特性逐步逼近原生性能。然而，尽管语言层面已成熟，WebAssembly 在 Web 平台中仍是“二等公民”，缺乏与 Web API 和加载机制的无缝集成，导致开发体验不及 JavaScript，限制了其广泛应用。作者提出通过 WebAssembly Components 和 esm-integration 等提案，简化模块加载并增强与 Web 平台的直接交互，从而提升其在 Web 中的“第一等”地位。

**BuildKit: Docker's hidden gem that can build almost anything**
> 文章介绍了 **BuildKit** 作为 Docker 背后的通用可插拔构建引擎，不仅能构建 OCI 镜像，还可输出任意文件系统产物（tar、目录、软件包等），核心是将构建定义转化为 **LLB**（低级构建描述），并通过内容寻址和并行执行实现高效缓存。其前端层可自定义语法（不限于 Dockerfile），结合灵活的输出类型，使 BuildKit 能支持多种非镜像构建场景，推动了像 Earthly、Dagger 等基于 LLB 的工具生态，展现出极高的可扩展性与技术创新价值。
