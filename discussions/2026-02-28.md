## 第二轮讨论：内容评选

### 文章摘要

**Accidentally disabling SSH access via scp**
> 这篇文章讲述了作者在使用 `scp -r` 将本地目录（权限为 777）递归复制到服务器主目录时，意外导致服务器上主目录权限也变为 777，从而触发 OpenSSH 的安全检查，拒绝所有基于公钥的登录。技术要点在于：OpenSSH 不仅对私钥文件权限敏感，也会检查服务器端用户主目录的权限与所有权，权限过于开放会直接阻止认证。创新价值在于通过重现实验定位问题根源，并提示在文件传输或同步工具操作时，需谨慎处理目录权限，否则可能导致远程访问中断。

**We deserve a better streams API for JavaScript**
> 文章指出现有的 WHATWG Web Streams API 虽然实现了跨浏览器与服务器的统一标准，但其设计受限于十年前的语言特性，导致在现代 JavaScript 中存在使用复杂、性能低下等问题。作者提出一种基于现代语言原生特性的新方案，在多个运行时的基准测试中性能提升可达 2–120 倍，显示通过重新设计 API 可以显著提高易用性与效率。

**Making WebAssembly a first-class language on the Web**
> 文章回顾了自 2017 年以来 WebAssembly 的功能扩展（如 SIMD、GC 支持、多返回值等），并指出尽管性能和语言特性已接近原生，但在 Web 平台上仍是“二等语言”，缺乏与 Web API 的紧密集成。当前 WebAssembly 的加载和使用 Web API 依赖 JavaScript，导致开发体验不佳，限制了普遍应用。作者提出通过 **WebAssembly Components** 和 **ESM 集成提案**简化加载与调用流程，从而提升其在 Web 平台的“第一等”语言地位。
