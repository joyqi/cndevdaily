### 文章摘要

**Free IP Geolocation Database**
> 这篇文章介绍了 **Free IP Geolocation Database**，一种每天更新、完全免费的 MMDB 格式 IP 地理定位数据库，包含 ASN、国家及大陆信息，支持 IPv4/IPv6，且无需许可证即可在任何 MMDB 兼容环境中直接使用。它的技术价值在于丰富的地理数据（包括组织名称、ISO 国家和大陆代码）与广泛语言支持（Python、Go、Node.js等），可无缝集成到现有工作流中，并采用 CC BY 4.0 许可，适合个人或商业项目。创新点在于**零使用限制、每日构建更新**，确保数据始终最新准确，且提供开源兼容性，降低接入成本。

**JSON Documents Performance, Storage and Search: MongoDB vs PostgreSQL**
> 文章比较了 **MongoDB**（原生面向文档）与 **PostgreSQL**（关系型为主，后续增强 JSON/JSONB 支持）在存储、索引和查询 JSON 文档方面的能力与性能差异。MongoDB 依赖灵活的无模式文档结构，优化于嵌套数据与多类型字段；PostgreSQL 则在保持关系型特性的同时，通过 JSONB 格式和多样化索引机制，在性能与功能上逐步接近甚至在部分场景竞争力相当。文章通过统一的数据结构与测试方案，对两者的 JSON 性能做了量化对比，为选择通用型数据库（PostgreSQL）还是专用型文档数据库（MongoDB）提供了依据。

**Show HN: I built a sub-500ms latency voice agent from scratch**
> 作者从零构建了一个端到端延迟低于 500 毫秒的语音代理，性能比现成平台（如 Vapi）快一倍，仅用一天和约 100 美元 API 费用完成。技术创新点在于手动实现语音代理的核心“轮换循环”，将 STT、LLM、TTS 通过流式管道高效协同，并优化模型选择与地理部署以减少延迟。文章强调语音代理的关键难点是持续实时的多模型编排与精准的说‑听状态切换，这种自主实现方式让开发者对延迟和体验的控制更精细化。
