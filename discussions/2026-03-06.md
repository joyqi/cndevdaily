**A GitHub Issue Title Compromised 4,000 Developer Machines**
> 这起 “Clinejection” 攻击事件利用了一个极具创新性的漏洞链：攻击者在 GitHub Issue 标题中嵌入 prompt injection，通过 Cline 项目的 AI 自动分流机器人执行恶意指令，进而实施缓存污染、凭证窃取，最终在 npm 发布被篡改版本并在 8 小时内入侵约 4,000 台开发者机器。该案例首次展示了**AI 工具可在无人工干预下“安装”另一 AI 代理**并获得系统控制，揭示了 AI 集成开发流程中的新型供应链风险。

**Building a Database on S3**
> 这篇文章回顾了 2008 年一篇在 Amazon S3 上构建数据库的论文，核心创新是**将存储与计算解耦**，通过 S3 作为 Page Store、SQS 作为日志存储（WAL），实现了类似现代无服务器和云原生数据库（如 Aurora）的架构雏形。系统利用**异步日志应用和检查点机制**来掩盖高延迟，同时通过幂等日志和原子提交协议应对 SQS 的非 FIFO、消息丢片等限制。文章展示了在“哑”云存储服务之上实现完整数据库所面临的工程复杂度，以及早期云数据库设计对后续架构理念的启发作用。
