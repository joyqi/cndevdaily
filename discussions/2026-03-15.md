## 第二轮讨论：内容评选

### 文章摘要

**Getting started with Claude for software development**
> 这篇文章面向想在 2026 年开始使用 Claude 等大语言模型（LLM）进行软件开发的程序员，强调其学习曲线类似于 Vim——初期不易上手，但投入时间可获得高回报。作者指出，要高效使用 LLM 需要有意图地、批判性地试验和总结经验，并保持积极、开放的心态；本篇作为系列的第一篇，主要铺垫背景并介绍上手的第一步，后续会逐步深入。

**Lies I was Told About Collaborative Editing, Part 2: Why we don't use Yjs**
> 文章主要观点是：作者反驳了“必须使用 Yjs 或其他 CRDT 算法才能实现高质量协作编辑”的常见说法，指出这些流行算法在离线和在线场景下都存在不适合的问题，并展示了一个仅需约 40 行代码、基于 prosemirror-collab 的简单方案，可以在有权威节点的条件下实现乐观更新、断网编辑、细粒度编辑追踪等功能。该方案技术价值在于极大简化实现复杂度、易于维护，同时保留离线同步等核心能力，唯一不足是无法支持真正的无主 peer-to-peer 编辑。

**Postgres with Builtin File Systems**
> 这篇文章介绍了 **DB9** ——一种将完整的 PostgreSQL 数据库与云文件系统融合的工具，专为 AI agents 提供统一的 SQL 查询与文件操作接口。它内置向量搜索、自动嵌入、HTTP 调用、分支环境复制、分布式定时任务以及类型生成，无需额外配置即可在终端快速创建和管理数据库。技术创新点在于将结构化数据与非结构化文件在同一工作空间中处理，并把 AI 常用的向量检索与外部 API 访问直接集成到 SQL 层，替代独立的向量数据库和数据管道，显著简化 AI 应用的存储与检索流程。
