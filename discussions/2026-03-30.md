### 文章摘要

**Free, native RISC-V (RV64GC) CI on GitHub**
> RISE 推出了免费的 **原生 RISC-V (RV64GC) GitHub Actions Runner** 服务，让开源项目无需模拟器即可在真实 RISC-V 硬件上运行 CI 测试，解决软件与硬件互相依赖的瓶颈问题。该服务通过安装 GitHub App 和简单修改 workflow，即可在独立 RISC-V 节点上执行任务，提供与官方 Runner 接近的环境并支持 Docker 构建。平台完全开源，并结合 Kubernetes 设备插件与单任务调度机制，确保性能稳定，显著降低开发者获取和维护 RISC-V 硬件的门槛。

**pretext: Pure JavaScript/TypeScript library for multiline text measurement & layout**
> Pretext 是一个纯 JavaScript/TypeScript 的多行文本测量与布局库，通过自定义文本测量逻辑避免 DOM 测量和浏览器重排，大幅提升性能与跨语言支持（包括 emoji 和混合双向文本）。它采用分离的 **prepare** 与 **layout** 流程，前者完成一次性文本分析与宽度缓存，后者在纯算术中快速计算高度和行数，使虚拟化 UI、精准布局和无布局偏移成为可能，并支持在 DOM、Canvas、SVG、WebGL 甚至服务端渲染。

**Making HNSW actually work with WHERE clauses**
> 文章核心内容总结如下：  

DuckDB 的默认向量搜索在执行 WHERE 过滤时采取“后过滤”，导致在低选择性场景下结果数量不足甚至为零。**hnsw_acorn** 通过 **ACORN‑1** 技术将过滤条件直接推入 HNSW 图遍历流程，并引入两跳扩展策略，显著提升 1%-60% 选择性下的召回率，同时根据选择性自动切换到标准遍历或顺序扫描以优化性能。配合 **RaBitQ** 量化可将索引内存占用减少 21-30 倍，并支持元数据 JOIN 与分组 Top‑K的 SQL 优化重写，实现透明、高效、标准 SQL 的向量搜索。
