### 文章摘要

**Claude Code Found a Linux Vulnerability Hidden for 23 Years**
> 文章介绍了研究员 Nicholas Carlini 使用 Anthropic 的 AI 工具 **Claude Code** 在 Linux 内核中发现多个远程可利用漏洞，其中一个隐藏了 **23 年** 未被发现。Claude Code 仅凭自动扫描源代码就能识别复杂漏洞，展现了其在安全分析中的**自主性与代码理解能力**。最具代表性的例子是它在 **NFS 驱动**中找到一个可远程读取内核内存的漏洞，证明大型语言模型能在深层协议与内核级安全研究中发挥创新作用。

**OpenScreen is an open-source alternative to Screen Studio**
> OpenScreen 是一个基于 **Electron、React、TypeScript 等技术栈**开发的开源屏幕录制工具，旨在提供 **免费、可修改和可商用的 Screen Studio 替代方案**。它支持屏幕录制、自动或手动缩放、音频采集、注解、裁剪和导出多分辨率视频等基础功能，重点突出 **简单易用与自由定制**。这一项目的技术创新在于以轻量化架构实现主流录屏核心功能，并开放源代码供社区协作与二次开发。

**AWS Engineer Reports PostgreSQL Perf Halved by Linux 7.0, Fix May Not Be Easy**
> AWS 工程师发现，Linux 7.0 的内核改动导致 PostgreSQL 在 Graviton4 平台上的吞吐量下降约一半，问题源于内核调度器简化后仅保留了部分抢占模式，增加了用户态自旋锁等待时间。尽管有补丁建议恢复 `PREEMPT_NONE` 默认配置，但内核维护者认为应由 PostgreSQL 适配新的 Restartable Sequences（RSEQ）time slice 扩展来解决。该事件凸显了 Linux 7.0 在调度机制上的重大调整及其对高并发数据库系统性能的影响。
