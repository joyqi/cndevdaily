## 第二轮讨论：内容评选

### 文章摘要

**OpenSSH begins warning for non-PQC key exchanges**
> OpenSSH 从 10.1 版本开始，当连接未使用抗量子密钥交换算法时会主动发出安全警告，以提醒用户可能面临“存储后解密”攻击风险。该项目自 9.0 版起已支持后量子算法（如 sntrup761x25519-sha512、mlkem768x25519-sha256），并在 10.0 版将后者设为默认，用以抵御未来量子计算机对传统加密的威胁。此更新体现了 OpenSSH 在量子安全方向的前瞻性布局和升级策略。

**Multi-agentic Software Development is a Distributed Systems Problem (AGI can't save you)**
> 本文将多智能体软件开发问题形式化为一个**分布式系统协调问题**，强调其复杂性并非可由更强大的模型（如AGI）自动消除。作者提出应开发新的**形式语言与编程工具**来描述和验证多代理协作过程，其中“舞蹈式（choreographic）语言”可作为优雅的建模形式。核心创新在于将LLM间的协作视为**分布式一致性问题**，并以理论模型证明：无论模型多智能，协调的不可解性仍然存在。

**defuddle: Get the main content of any page as Markdown**
> Defuddle 是一个用于提取网页主要内容的工具，可自动清除评论、侧边栏、页眉页脚等冗余信息，仅保留正文，并支持输出为 HTML 或 Markdown。它可在浏览器扩展（如 Obsidian Web Clipper）或 Node.js 环境中使用，支持 CLI 调用，功能包括结构化元数据提取、脚注与代码块一致化处理及基于移动端样式的内容识别。相比 Mozilla Readability，Defuddle 在容错性、格式统一性及元数据提取能力上更具优势。
