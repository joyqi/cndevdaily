### 文章摘要

**Applying "Programming Without Pointers" to an mbox indexer using Zig**
> 本文介绍作者在使用 Zig 编写 mbox 差异计算工具时，如何应用“Programming Without Pointers (PWP)”理念来提升性能。核心创新在于通过减少内存分配与指针管理，将 mbox 文件索引从每条消息单独分配内存改为集中式字符串存储，从而显著降低复杂度与内存开销。该实践展示了 Zig 在高性能、低分配场景下的设计优势。

**Under the hood of MDN's new frontend**
> MDN 新前端最核心的创新在于重构其架构，使之更易维护、更高效。团队用 Web Components 替代旧 React 架构，从根本上解决了样式混乱、技术债高和构建流程复杂等问题，实现了模块化、原生化、可复用的组件体系。同时，新方案显著优化了前端性能与可扩展性，为未来内容交付和交互升级奠定技术基础。

**AWS Engineer Reports PostgreSQL Performance Halved By Linux 7.0, But A Fix May Not Be Easy**
> AWS 工程师发现，Linux 7.0 内核使 PostgreSQL 吞吐量在 Graviton4 上下降约 50%，问题源于新版本限制了内核可用的抢占模式，增加了用户态自旋锁的开销。虽然有补丁提议恢复 PREEMPT_NONE 默认模式，但内核维护者倾向于让 PostgreSQL 采用新的 RSEQ（Restartable Sequences）时间片扩展来解决性能问题。这意味着未来性能优化或将转向数据库自身适配 Linux 7.0 的调度机制，而非回退内核特性。
