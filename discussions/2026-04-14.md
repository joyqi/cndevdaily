### 文章摘要

**Lean proved this program was correct; then I found a bug**
> 这篇文章讲述作者在对经过 Lean 形式化验证的 zlib 实现（lean‑zip）进行模糊测试时，发现了 Lean 运行时中的堆缓冲区溢出漏洞，而验证代码本身没有问题。文章指出：即使形式化验证能证明算法逻辑正确，运行时与外围组件仍可能成为漏洞源。作者借此强调，随着 AI 自动化漏洞挖掘能力快速提升，传统软件安全成本正在急剧下降，结合形式化验证与自动化测试或将成为提升软件安全性的关键方向。

**GitHub Stacked PRs**
> GitHub **Stacked PRs** 提供原生支持和专用 CLI（`gh stack`），让开发者将大型变更拆分为一系列相互依赖、可独立审查的小型 PR，并可通过一键操作合并整个堆栈。该功能在 GitHub UI 中直观展示层次关系、自动处理级联 rebase，并支持与 AI 编码代理集成，显著提升代码审查效率与团队协作质量。

**App Store Reviews are Busted**
> 作者指出 App Store 的评分系统存在结构性缺陷：评分算法与用户意图严重错位，例如 4 星好评在统计上会拉低平均分，从而误伤优秀应用。由于评分直接影响搜索与推荐算法，开发者被迫优化“求评时机”甚至过滤非 5 星评分，导致系统反映的不是产品质量，而是评分机制利用水平。作者最终拒绝参与这种“评分博弈”，主张以产品本身价值取胜。
