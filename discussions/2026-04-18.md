## 第二轮讨论：内容评选

### 文章摘要

**Building ezli.me, a link shortener in Rust**
> 这篇文章介绍了作者用 **Rust** 构建自托管链接缩短服务 **ezli.me** 的设计与实现。核心创新在于极简高效的架构：通过 **axum + tokio + PostgreSQL + diesel** 实现轻量服务端，用 **内存缓存与批量异步写回机制** 避免请求路径中的数据库写操作，大幅提高吞吐和响应速度。短链接生成采用 **基于哈希的确定性 Base62 编码**，确保可重复且无状态，整体系统在极低资源占用下实现快速、安全、可自托管的链接重定向能力。

**Healthchecks.io Now Uses Self-hosted Object Storage**
> Healthchecks.io 将其对象存储从托管服务迁移到自建方案，使用 Versity S3 Gateway 结合 Btrfs 文件系统，以提升性能稳定性并降低复杂性与成本。文章详细比较了 AWS、OVHcloud、UpCloud 等托管存储在费用、性能和可靠性上的局限，最终展示自建存储在中等规模场景中具备高控制力、可维护性和更可预测性能的技术优势。

**zmx - run local code agents on remote machines**
> zmx 是一个零配置的命令行工具，可让本地代码代理在远程机器上安全执行命令并管理持久会话。新版本 v0.5.0 引入了 `run`、`write`、`tail` 等功能，通过流式传输命令和文件实现交互和可恢复的远程执行。其技术创新在于结合 libghostty 的终端恢复机制，实现无状态、可监视的远程任务运行，为人类用户与自动代码代理都提供高效的远程操作接口。
