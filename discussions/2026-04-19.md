## 第二轮讨论：内容评选

### 文章摘要

**How (and why) we rewrote our production C++ frontend infrastructure in Rust**
> 文章讲述了一家服务提供商将其关键生产级 C++ 前端基础设施（负责缓存、代理、路由、访问控制和 TLS 等核心功能）重写为 Rust 的过程与原因。团队选择 Rust 的主要动机是提升内存与并发安全性、改善代码可维护性，并利用 Rust 强大的生态（Cargo）简化开发和扩展。该迁移展示了 Rust 在高可靠性网络核心组件中的成熟度和替代 C++ 的潜力。

**How GitHub uses eBPF to improve deployment safety**
> GitHub 通过在部署系统中引入 **eBPF（Extended Berkeley Packet Filter）** 技术，实现了对部署脚本网络访问的细粒度监控与控制，从而防止部署过程中产生新的**循环依赖**。他们利用 **BPF_PROG_TYPE_CGROUP_SKB** 类型的 eBPF 程序，在特定 cGroup 内拦截网络出口调用，仅限制部署脚本访问外部资源而不影响生产流量。这种做法显著提升了部署安全性与故障恢复能力，并展示了 eBPF 在运维场景中创新的应用潜力。

**Show HN: MDV – a Markdown superset for docs, dashboards, and slides with data**
> MDV 是一个基于严格 CommonMark 的 Markdown 超集，扩展了 YAML front-matter、数据可视化代码块和 ::: 容器语法，使用户能在纯 Markdown 中编写文档、仪表板和幻灯片。其技术创新在于用极简语法直接生成带图表、KPI、布局的可视化文档，渲染为无 JS 的可移植 HTML/PDF，并提供 VS Code 实时预览与 CLI 工具。该项目简化了数据驱动报告的制作流程，实现了 Markdown 到数据可视化内容的一体化生成。
