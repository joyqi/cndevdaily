## 第二轮讨论：内容评选

### 文章摘要

**GitHub Actions is the weakest link**
> 这篇文章指出 GitHub Actions 已成为开源供应链安全的核心薄弱点。其问题在于默认设计仍基于企业私有仓库场景，却被广泛用于开放仓库，从而导致匿名分支和可写标签被攻击者利用，造成凭证泄露及恶意代码传播。作者强调技术根源在于 Actions 的依赖解析机制（动态解析可变 Git 标签、无锁定文件或哈希验证），呼吁对平台默认安全模型进行系统性重构，以提升供应链完整性。

**Bypassing DPI with eBPF, no VPN or proxy needed**
> 这篇文章介绍了利用 **eBPF** 实现的一种透明、轻量的 **DPI（深度包检测）绕过方法**，无需 VPN 或代理。其核心创新在于：  
1. **在 TCP 建立后立即发送伪造的 TLS ClientHello**（带虚假 SNI、极低 TTL），诱使 DPI 误判连接目标；  
2. **通过 eBPF sock_ops 控制窗口大小与 MSS** 逼迫真实 ClientHello 被分段，从而让 DPI 无法识别 SNI；  
3. **结合本地 DoH 解决 DNS 污染**，实现系统级、安全、自动的封锁绕过机制。

**Localsend: An open-source cross-platform alternative to AirDrop**
> LocalSend 是一款开源、跨平台的本地文件与消息传输工具，通过 REST API 和 HTTPS 通信实现端到端加密，无需互联网或第三方服务器即可在局域网中安全传输。其技术亮点在于完全自生成 TLS/SSL 证书，确保隐私与安全，同时采用 Flutter + Rust 架构实现多平台兼容与高性能运行。项目支持多平台打包与便携模式，方便用户部署与二次开发。
