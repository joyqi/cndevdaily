### 文章摘要

**Moving away from Tailwind, and learning to structure my CSS**
> 作者分享了从 Tailwind 迁移到语义化 HTML 与原生 CSS 的过程，探索如何建立系统化的 CSS 结构。文章强调 Tailwind 在体系化思维上的启发——如重置样式、统一配色与字体系统——并介绍了作者为组件化管理、命名规范、色彩与排版变量所制定的规则。技术创新点在于从工具依赖转向自定义 CSS 架构，借助 Tailwind 的理念构建更语义化、可维护的样式体系。

**SQL: Incorrect by Construction**
> 本文指出，SQL 在设计层面上使得并发错误容易出现，即使是看似合理的代码也可能导致资金丢失、超额提款或死锁等问题。作者通过分析一个转账示例，依次揭示了原子性缺失、TOCTOU 竞态条件以及死锁风险，并展示了如何通过事务封装、行级锁（如 `UPDLOCK`）和一致的锁顺序来修复这些问题。技术创新点在于强调：SQL 的语义并不自动保证并发安全，开发者需显式控制事务和锁才能实现正确的并行行为。

**What we learned using AI agents to refactor a monolith**
> 1Password 使用 AI agent 对数百万行的 Go 单体系统进行“智能重构”实验，探索了自动化分析与人类工程师协作的最优比例。团队构建了结合 Go SSA、SQL 解析和 DataDog 运行时数据的 agentic 工具链，实现了可解释且可复现的系统分解与依赖映射。技术创新在于将 AI 从“决策者”转变为“工具构建者”，以生成稳定的分析层，从而安全地指导大型生产系统的服务化与架构演进。
