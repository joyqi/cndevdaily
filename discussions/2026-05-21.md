**Staged publishing for npm packages**
> “Staged publishing for npm packages” 介绍了 npm 新增的分阶段发布机制：在发布包之前先进入一个“待审核区”，需由具备发布权限的维护者通过二次验证（2FA）进行批准后才能正式上线。该功能增强了包发布的安全性与可追溯性，支持 CLI 与 npmjs.com 双端操作，并可与 OIDC 的 Trusted Publishing 工作流结合，实现更安全、可控的持续集成发布流程。

**modulejail: Proactively shrink a Linux host's kernel-module attack surface by blacklisting every module not currently in use**
> **总结**：  
ModuleJail 是一个单文件 POSIX Shell 脚本，用于通过自动生成黑名单来禁用 Linux 主机上未使用的内核模块，从而主动缩小内核模块的攻击面。它无需守护进程或复杂依赖，运行一次即可生成定制化的 `/etc/modprobe.d/` 黑名单配置。其核心创新在于提供一种简单、可审计且立即可用的防御措施，让管理员在大规模内核漏洞披露潮中，通过最小化加载模块数量来显著降低潜在攻击风险并延缓应急补丁压力。
