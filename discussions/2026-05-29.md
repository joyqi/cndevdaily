## 第二轮讨论：内容评选

### 文章摘要

**GNOME 2.20 but its Web Components**
> 这篇文章介绍作者将个人网站改造成模拟 GNOME 2 桌面的实验项目。网站以 **Web Components** 和 **Lit** 框架构建，利用 **lit-ssr** 生成 DSD 模板，并结合 **View Transitions API** 实现页面间平滑的“窗口”切换。项目技术上展示了现代前端框架在重现早期桌面体验方面的创造性运用，同时兼具趣味性与可扩展性。

**Building durable workflows on Postgres**
> 本文提出一种基于 **Postgres 的耐久型工作流体系**，通过将工作流状态直接持久化到数据库，实现无需中央编排器的可靠分布式执行。与传统的外部编排方式（如 Temporal、Airflow）相比，这种模式让应用服务器通过 Postgres 自协作地调度与恢复任务，大幅简化系统架构。其创新点在于：**利用数据库自身的事务一致性与锁机制，实现任务分发、状态检查点与故障恢复的原子化管理**，进而以 Postgres 原生能力解决可扩展性、可用性与安全性问题。

**Endive: A JVM native WebAssembly runtime**
> Endive 是一个纯 JVM 实现的 WebAssembly 运行时，无需任何本地依赖或 JNI，可在任何支持 JVM 的环境中安全运行 Wasm 程序。它基于 Chicory 项目演化而来，目标是成为 JVM 上默认且最安全的 Wasm 运行时，实现全规范支持并与 Java 生态深度集成。其技术创新在于以 Java 实现 Wasm 解释与编译引擎，保留 JVM 的安全性与可移植性，同时解决了传统原生 Wasm 运行时在分发和运行层面的复杂性。
