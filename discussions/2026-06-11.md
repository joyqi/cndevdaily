## 第二轮讨论：内容评选

### 文章摘要

**How building an HTML-first site doubled our users overnight**
> 通过回归 HTML 优先的渐进增强架构，将复杂表单拆解为独立页面并由后端保存每一步数据，确保仅依靠基础 HTML 即可在任何设备与低质量网络下完整操作，从而一夜之间将用户量翻倍。

**An interactive introduction to the terrific experience of rendering Arabic typography and its technical debt**
> 本文以一个前端团队遇到的阿拉伯文网页排版缺陷为切入点，揭示了现代 CSS 提供的 `text-align: justify` 只能通过拉伸单词间距来实现两端对齐，却无法自动生成符合阿拉伯书法传统的「笔画内拉伸」（如 kashida），这一矛盾暴露出从旧版 Unicode 编码、字体整形引擎到 Web 排版标准之间的深层技术债务。文章的技术价值在于通过手工预置每个拉伸点的交互式对比，生动展示了真正古典阿拉伯排版所需的字形级精密控制，凸显了现有网页技术在复杂文字渲染上的局限性与仍需手工补救的创新实践。

**A line-by-line translation of the OCaml runtime from C to Rust**
> 作者利用Claude AI以逐行、逐文件的渐进方式，将OCaml运行时从C完全移植到Rust，并成功通过上游未修改的完整测试套件，实现了零C代码残留。技术价值在于验证了大语言模型在辅助移植包含多核机制与真实垃圾回收器的复杂底层运行时方面的可行性，其创新点在于采用人工引导下的精确翻译策略，确保编译器始终可运行，最终性能表现意外良好。
